# SocioSim Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build SocioSim — a reproducible, uncertainty-quantified, policy-as-code social-interaction simulator with US/EU/CN/FTC packs, moderation, ads, feed ranking, logging/replay, analytics, and validation, per the approved design doc (`docs/superpowers/specs/2026-06-12-sociosim-design.md`).

**Architecture:** Modular Python package; columnar NumPy agent state for vectorized ticks; per-event logic for policy/moderation/ads; JSONL event-sourced logs with manifest-based deterministic replay; pluggable content generation (deterministic templates default, cached Claude adapter optional).

**Tech Stack:** Python 3.11+, numpy, networkx, scipy, pyyaml, pytest. Optional extras: matplotlib (figures), anthropic (LLM adapter).

**Run profile defaults (approved):** standard = 10,000 agents × hourly × 28 days × 100 replicates; quick = 1,000 × 7 days; tests use ≤200 agents × ≤48 ticks.

### Task 5: agents (personas, diurnal activity, state)
**Files:** Create `socio_sim/agents/personas.py`, `socio_sim/agents/state.py`; Test `tests/test_agents.py`.
- [ ] Columnar persona arrays sampled from documented distributions: `age_group` (categorical incl. "13-17" minors ~8%), `is_minor`, `ideology` (2D normal clipped [-1,1]), `trust` Beta(2,2), `activity` lognormal heavy tail normalized, `ad_responsiveness` Beta(2,5), `moderation_attitude` uniform, `influencer` flag = top-degree 1%, `vulnerable` flag.
- [ ] Diurnal curve: 24-vector with peak 16–18h, trough 4–6h (cited circadian lit), normalized; `p_active(tick) = activity * diurnal[hour]` clipped ≤1; vectorized Bernoulli draw of active mask.
- [ ] `AgentState`: beliefs (per-topic float array), fatigue, ad_exposures counter, last_seen; vectorized `update_beliefs(exposed_topic_scores, trust, lr)` (convex shift toward content stance scaled by trust), `decay_fatigue`.
- [ ] Tests: minors flagged consistently, diurnal peak/trough ordering, activity heavy tail, belief update direction + bounds. Commit.

### Task 6: content (items, template generator, noisy classifier, CN labels, Claude adapter)
**Files:** Create `socio_sim/content/items.py`, `socio_sim/content/generate.py`, `socio_sim/content/classify.py`, `socio_sim/content/claude_adapter.py`; Test `tests/test_content.py`.
- [ ] `ContentItem`: id, author_id, tick, media_type (text|image|video), topic, stance (-1..1), text, true_categories (set from: hate, harassment, fraud, misinfo, adult, illegal_goods, self_harm, political, ad, sponsored, ai_generated), ai_generated flag, explicit_label (bool), implicit_watermark ({provider, content_ref} | None), disclosure_present (bool).
- [ ] `TemplateGenerator(rng)`: persona/topic-conditioned seeded templates; assigns ground-truth categories with configurable base rates (e.g., misinfo 2%, hate 1%); AI-generated items get watermark fields per CN rules when CN pack active.
- [ ] `NoisyClassifier(targets, rng)`: per-category Bernoulli corruption to hit configured precision/recall; outputs per-category scores in [0,1]; `confusion(true, pred)` matrix helper.
- [ ] `ClaudeAdapter(cache_path)`: protocol-compatible with TemplateGenerator; if `anthropic` importable and key set → call, write JSON cache keyed by (prompt_hash); on cache hit read; on failure emit `degradation` event and delegate to TemplateGenerator. No network in tests (cache-injected).
- [ ] Tests: empirical precision/recall within ±0.05 of targets at n=20k, watermark presence iff ai_generated under CN, adapter fallback + cache determinism. Commit.

### Task 7: policy (rule packs + engine)
**Files:** Create `socio_sim/policy/engine.py`, `socio_sim/policy/packs/us_section230.yaml`, `eu_dsa.yaml`, `cn_ai_label.yaml`, `ftc.yaml`; Test `tests/test_policy.py`.
- [ ] YAML rule schema exactly per Spec §3.6: `rule_id, jurisdiction, trigger {categories, min_severity, user_type?, conditions?}, action, notice_required, appeal_allowed, deadline_hours, evidence_threshold, log_required, priority`. Loader validates schema, records pack version string.
- [ ] Engine `evaluate(item, scores, context)` → ordered `PolicyDecision` list (rule_id, action, notice_required, appeal_allowed, deadline_hours, rationale string with thresholds/scores). Composition: highest-priority terminal action (remove > block > downrank > label) plus all side obligations (notices, logging). Severe categories (self_harm, illegal_goods) with no matching rule → `policy_gap` escalation (fail closed).
- [ ] Pack content: US — Good-Samaritan removals immune flag, criminal/IP/privacy carve-outs escalate; EU — illegal-content removal w/ notice+appeal+24h deadline, easy-flag handling, ad rules (no minors targeting, no sensitive-data targeting), non-personalized feed right; CN — unlabeled synthetic media gets platform label added + provider obligations + `log_retention_months: 6`; FTC — sponsored content without disclosure → insert disclosure / flag violation.
- [ ] Tests: per-pack semantics (table-driven), composition priority, fail-closed gap, notice/appeal flags propagate. Commit.

### Task 8: moderation (workflow, escalation, appeals)
**Files:** Create `socio_sim/moderation/workflow.py`; Test `tests/test_moderation.py`.
- [ ] `ModerationQueue`: intake from classifier scores ≥ rule evidence_threshold or user flags; apply policy decisions; escalation queue with simulated human review (configurable accuracy/delay, seeded); appeals: filed w.p. depending on moderation_attitude; appeal review reverses FPs w/ configured probability; deadline tracking → `deadline_missed` data on event.
- [ ] All actions logged with `decision_rationale` (rule ids + scores + thresholds; structured, no CoT).
- [ ] Tests: flag→review→action pipeline, appeal reversal restores content, deadline accounting, human-review determinism. Commit.

### Task 9: ads (auction, constraints, disclosure, holdouts, calibration)
**Files:** Create `socio_sim/ads/campaigns.py`, `socio_sim/ads/auction.py`, `socio_sim/ads/measure.py`; Test `tests/test_ads.py`.
- [ ] `Campaign`: id, advertiser, bid, budget, targeting (age groups, interests, sensitive flags), creative (ContentItem with ad category, sponsored flag), holdout_fraction.
- [ ] Second-price auction per slot among eligible campaigns: eligibility = targeting match ∧ policy constraints (EU: exclude minors entirely from targeted ads; sensitive-attribute targeting stripped) ∧ budget ∧ frequency cap (default 4/day/agent) ∧ fatigue multiplier. Winner pays second price; budget decremented.
- [ ] FTC disclosure: sponsored/influencer creatives get "#ad — paid partnership" appended when `ftc_compliance=True`; toggle off for counterfactual runs → `disclosure_present=False` and policy pack flags violations.
- [ ] Holdout: per-campaign agent-level assignment (seeded hash) — holdout agents never see that campaign; `measure.py` computes CTR/CVR/CPM/CPC, incremental lift (exposed vs holdout conversion), ROI; Beta-Binomial posterior (prior from benchmark CTR) → 95% credible intervals.
- [ ] Click model: p_click = base_ctr × ad_responsiveness × fatigue × creative match; conversion downstream w.p. cvr.
- [ ] Tests: second-price property (winner pays runner-up bid), minors excluded in EU mode, frequency cap binds, holdout never exposed, posterior interval covers true rate in simulation, disclosure insertion. Commit.

### Task 10: feed (strategies, bandit, EU opt-out)
**Files:** Create `socio_sim/feed/ranking.py`; Test `tests/test_feed.py`.
- [ ] Candidate pool = recent posts from followees + bandit exploration slice from network. Strategies: `chronological` (recency only), `random`, `personalized` score = w·[recency_decay, affinity (past engagements w/ author), predicted_engagement (topic-belief dot), dwell_proxy, trust]; weights configurable per content category; downranked items multiplied by penalty; removed items excluded.
- [ ] Epsilon-greedy exploration (default ε=0.1) injecting unseen-author posts.
- [ ] EU mode: agents with personalization opt-out (config rate) always get chronological. Every impression logs features + scores (auditability).
- [ ] Ads interleaved at fixed slots (e.g., every 5th position) from auction winners.
- [ ] Tests: chronological ordering, personalization beats random on affinity alignment, opt-out honored, downrank lowers position, impression logging completeness. Commit.

### Task 13: validation (targets, KS, history matching, ABC, sensitivity)
**Files:** Create `socio_sim/validation/targets.py`, `socio_sim/validation/calibrate.py`, `socio_sim/validation/sensitivity.py`, `data/benchmarks/default_targets.json`; Test `tests/test_validation.py`.
- [ ] `default_targets.json`: named aggregate targets w/ source notes (degree tail exponent ~2–3, clustering 0.1–0.3, diurnal peak/trough hours, daily-post rate distribution, ad CTR ~0.5–2%, appeal-grant ~10–40%) + tolerance per target.
- [ ] KS distance between simulated and target samples; `implausibility(params)` = max standardized discrepancy; history matching: LHS over param ranges → keep non-implausible (I < 3); ABC rejection: among survivors accept ε-closest → empirical posterior + credible intervals.
- [ ] Sensitivity: first-order variance-based indices via correlation-ratio estimator over LHS design (documented as Sobol-style approximation), ranked parameter report.
- [ ] Monte Carlo runner: replicate sweep (params from posterior or priors) → outcome distributions median + 2.5/97.5 percentiles.
- [ ] Tests (small n, marked slow where needed): implausibility monotone in discrepancy, ABC recovers planted parameter on toy model, sensitivity ranks the dominant toy parameter first. Commit.
